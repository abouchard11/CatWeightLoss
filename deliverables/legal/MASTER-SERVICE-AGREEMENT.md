# ARTICLE 1: DEFINITIONS

## 1.1 Definitions

**"Activation"** means a unique instance of a Consumer launching the Application via QR Code deep link and completing the initial brand splash screen.

**"Application"** means the Cat Weight Companion iOS mobile application, including all updates, modifications, and enhancements made by Licensor during the Term.

**"Brand Configuration"** means the customized visual and functional parameters applied to the Application for Licensee, including but not limited to: brand name, primary color, accent color, and associated SKU data.

**"Completion"** means an Activation that results in the Consumer successfully completing the Quick Setup flow and entering the Application dashboard.

**"Consumer"** means an end-user who downloads and uses the Application.

**"Critical Bug"** means a defect that renders the Application inoperable, causes data loss, or prevents core functionality from operating as specified.

**"Deliverables"** means the Application configured with Licensee's Brand Configuration, QR Code specifications, and associated documentation.

**"Engagement Metrics"** means anonymized, aggregated data regarding Consumer interactions with the Application, as further described in Exhibit B.

**"Non-Critical Bug"** means a defect that does not constitute a Critical Bug but adversely affects Application functionality or user experience.

**"QR Code"** means the quick response code containing the deep link URL that launches the Application with Licensee's Brand Configuration.

**"SKU"** means a specific product stock-keeping unit of Licensee's pet food products that will be associated with a QR Code.

**"Source Code Escrow"** means the arrangement described in Article 10 whereby Application source code is held by a neutral third party.

**"Term"** has the meaning set forth in Section 11.1.

# EXHIBIT B: ENGAGEMENT METRICS SPECIFICATION

## Metrics Collected

All metrics are aggregated and anonymized. No personally identifiable information is collected or stored.

| Metric | Definition | Granularity |
|--------|------------|-------------|
| Activations | Count of QR Code deep link launches | Per SKU, Daily |
| Completions | Count of Quick Setup flow completions | Per SKU, Daily |
| App Opens | Count of app launches post-setup | Per SKU, Daily |
| Weight Entries | Count of weight log entries | Per SKU, Daily |
| 3+ Entry Users | Count of users with 3+ weight entries | Per SKU, Weekly |
| Progress Bucket | Distribution of users by progress % (0/25/50/75/100) | Per SKU, Weekly |

## Derived Metrics (Calculated)

| Metric | Formula |
|--------|---------|
| Completion Rate | Completions / Activations |
| 3+ Entry Rate | 3+ Entry Users / Completions |
| Avg Days Between Logs | Sum(days between entries) / Count(entries) |

## Data Not Collected

The following data is intentionally **NOT** collected:

| Excluded Data | Reason |
|---------------|--------|
| Actual weight values | Medical liability |
| Individual user journeys | Privacy |
| Feeding compliance | Blame risk |
| Cat names or photos | PII |
| Location data | Privacy |
| Device identifiers (raw) | Privacy — only hashed |

## Data Retention

- Engagement Metrics: 90 days, then automatically purged
- Device hash: Non-reversible, changes on app reinstall
- No data transmitted off-device at any time

## Reporting

Licensor shall provide Licensee with:

- Weekly metrics summary via email
- Monthly executive dashboard (PDF)
- Raw data export upon request (CSV format)
